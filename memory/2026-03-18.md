### Key Learnings About Subagent Spawning

**What worked:**
- Pre-reading files and having subagents write implementation code works well
- Files get written in 1-4 minutes when task is well-scoped
- MiniMax M2.7 handled technical TypeScript writing successfully

**What failed (consistently):**
- tester subagents for rl_hooks_test_suite.ts timed out 3x (5min, 5min, 10min)
- Pattern: subagent gathers 50k-214k tokens but produces no output
- Pre-loading file contents into prompt DID NOT fix the issue
- All 3 testers showed same pattern: high token count, zero file output

**Bottleneck insight:**
- Pre-loading didn't solve it — bottleneck isn't reading
- Complex multi-file test suite writing exceeds subagent capability in one shot
- Model appears to spend tokens "thinking" but never transitions to writing

**Pattern for future:**
- Implementation files: spawn subagent with interfaces pre-loaded → works ✅
- Large complex test files: write myself, or break into tiny pieces (one test at a time)

## HOOK.md Format Fix (Evening Session)

### Problem
All 13 managed hooks showed blank descriptions in `openclaw hooks list`. Bundled hooks (boot-md) showed descriptions correctly.

### Root Cause
I used wrong format in HOOK.md frontmatter:
```yaml
# WRONG (what I used):
name: biascheck-gym
icon: 🔍
triggers:
  - think:end
```

OpenClaw expects:
```yaml
# CORRECT:
name: biascheck-gym
description: "Reward bias detection for PPO/CFR training"
metadata:
  {
    "openclaw": {
      "emoji": "🔍",
      "events": ["think:end"]
    }
  }
```

### Fix Applied
Rewrote all 13 HOOK.md files with correct `metadata.openclaw.emoji` and `metadata.openclaw.events` format.

## OpenClaw Hook Loader Architecture (Critical Learning)

The OpenClaw hooks loader:
- Loads `handler.ts` as entry point
- **No TypeScript compilation** — expects plain JavaScript
- **No ESM multi-file imports** — single file only
- Works hooks: darwin-swarm, biascheck-gym (single-file, no imports)
- Broken hooks: rl-harness (had multi-file TS imports)
- Bundled hooks use simple JS with YAML frontmatter for metadata

**Pattern for future hooks**: All-in-one handler.ts, no separate module files.

## [CRITICAL BREAKTHROUGH] Managed Hook Loading Issue SOLVED (Apr 4, 2026)

### The Problem
After WEEKS of failing with "Handler 'default' from meta-gym is not a function", we finally discovered the root cause.

### The Discovery
The issue was in how we exported the handler function:

```javascript
// ❌ WRONG - caused "Handler 'default' is not a function"
module.exports = { handler };

// ✅ CORRECT - export the function DIRECTLY
module.exports = handler;
```

The OpenClaw hook loader expects `module.exports` to BE the function itself, not an object containing a handler property.

### Additional Issue: ESM vs CommonJS
Using `.ts` files with ESM `import/export` syntax causes "Unexpected token 'export'" errors because `package.json` says `"type": "commonjs"`. Solution: use plain `.js` files with CommonJS syntax.

### The Fix Applied
1. Changed `handler.ts` → `handler.js`
2. Changed ESM imports → CommonJS `require()`
3. Changed `export default handler` → `module.exports = handler`
4. Changed `module.exports = { handler }` → `module.exports = handler`

### Golden Rules for Managed Hooks
1. **Two files only:** `HOOK.md` + `handler.js`
2. **CommonJS syntax:** `require()` and `module.exports = handler`
3. **Export function directly:** `module.exports = handler` (the function itself, NOT { handler })
4. **No ESM in .js files:** Don't use `import` or `export`
5. **Debug with console.log:** Add logging to verify loading
6. **Return modified ctx or meta:** Return `{}` to pass through unchanged

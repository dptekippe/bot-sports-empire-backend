## [PROJECT] Photo Compression — Research Done

**Daniel's idea:** Method to shrink photo file sizes (for his mom's iPhone). Like zip files but for photos. A new way to map every pixel so it can be recreated. Could enable exact AI recreation of photos.

**Research conducted via Exa web search:**

### Key Findings

**1. Fractal Compression (Barnsley, 1988)**
- Almost EXACTLY what Daniel described
- Stores mathematical TRANSFORMATIONS (IFS), not pixels
- Natural images have self-similarity → parts resemble other parts
- Compression ratios: 10:1 to 170:1
- Resolution-independent decompression
- Problem: Encoding is computationally expensive (brute force search)
- Today: Largely abandoned in favor of neural methods

**2. Overfitted Neural Codecs (2024-2025)**
- Fits a SMALL model to EACH individual image (not one model for all)
- C3 (Google): Matches H.266 at low complexity
- FNLIC (CVPR 2025): Pre-fits prior, then overfits to each image
- MoRIC: Modular region-based compression
- These are lossless or near-lossless. Real compression ratios.

**3. AI Exact Recreation Problem**
- Daniel nailed the issue: Diffusion models (Midjourney, DALL-E) are PROBABILISTIC — exact pixel recreation is impossible
- Meta's PerCo uses diffusion at 0.003 bpp but output is GENERATIVE not exact
- Fundamentally at odds with how generative AI works

**4. The "Map Every Pixel" Insight**
- If you truly map every pixel exactly → you're storing the original data → not compression
- Compression comes from finding a MORE EFFICIENT representation than raw pixels

### Daniel's Practical Use Case
Help his mom compress iPhone photos locally (not cloud). Suggested approach:
- App on Mac that selects photos
- Compresses using overfitted neural codec (FNLIC-style)
- Stores compressed versions locally
- Restores to iPhone when needed

| Overfitted neural codec | No (2024-2025) | Yes | Actually works, real compression |

---

#### 2. H.261 Video Codec — **FREE**

- First video codec standard (1988)
- All patents expired
- No licensing required
- Foundation for modern video codecs
- Open source in FFmpeg

---

#### 3. H.263 Video Codec — **FREE**

- Low-bitrate video compression (1995)
- Baseline patents: all expired
- Later additions: expired 2020-2023
- **Now fully royalty-free**

**Used by:** Early video phones, RealVideo, foundation for MPEG-4 Part 2

---

#### 4. JPEG 2000 (Wavelet Compression) — **Partially Free**

- Uses Discrete Wavelet Transform (DWT) instead of DCT
- Better quality than JPEG at same compression
- Lossless option + progressive decoding
- Part 1 (core): Royalty-free from 17 patent holders
- Extensions: some patent concerns remain

**Why it never took off on web:**
- Required more memory (2000: avg PC had 64MB)
- Not supported by browsers
- JPEG was "good enough"

**Where used TODAY:** Medical imaging (DICOM), geospatial/satellite, digital cinema (DCP), forensic science, PDF/A files
